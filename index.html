Conceitos Fundamentais

Entenda os pilares tecnológicos que tornam possível a transformação de APIs em ferramentas inteligentes

Inteligência Artificial (IA)

A IA é a capacidade de máquinas simularem a inteligência humana, processando informações, aprendendo padrões e tomando decisões. No contexto deste projeto, utilizamos modelos de linguagem avançados (LLMs) como GPT-4 que podem compreender linguagem natural e executar tarefas complexas através de instruções textuais.

Agentes de IA

Agentes são sistemas de IA autônomos que podem perceber seu ambiente, tomar decisões e executar ações para atingir objetivos específicos. Eles podem usar ferramentas externas, manter memória de conversas e raciocinar sobre problemas complexos, funcionando como assistentes inteligentes especializados.

Spring Framework

Spring é um framework Java robusto e maduro para desenvolvimento de aplicações empresariais. O Spring AI é uma extensão moderna que facilita a integração de capacidades de IA em aplicações Spring, oferecendo abstrações para modelos de linguagem, embeddings e ferramentas de IA.

OpenAPI/Swagger

OpenAPI (anteriormente Swagger) é uma especificação padrão para descrever APIs REST. Ela define endpoints, parâmetros, tipos de dados e respostas de forma estruturada e legível por máquinas. Isso permite documentação automática, geração de código e, no nosso caso, criação dinâmica de ferramentas para IA.

Como o Swagger Agent Funciona

Um processo automatizado que transforma especificações de API em ferramentas inteligentes

1

Leitura de Especificações

O sistema varre recursivamente todas as pastas em busca de arquivos swagger.json, permitindo organização por projetos e microserviços.

2

Parsing e Análise

Cada especificação OpenAPI é analisada para extrair endpoints, parâmetros, schemas de request/response e metadados importantes.

3

Geração de Ferramentas

Para cada endpoint, uma ferramenta dinâmica é criada com nome único, descrição detalhada e função executável que a IA pode chamar.

4

Integração com IA

As ferramentas são registradas no agente de IA, que recebe um prompt otimizado explicando como e quando usar cada ferramenta.

5

Execução Inteligente

Quando o usuário faz uma pergunta, a IA analisa a solicitação, escolhe as ferramentas apropriadas e executa as chamadas de API necessárias.

Exemplo: API de Cartões

Transformação de um endpoint REST em ferramenta de IA

1. Especificação OpenAPI Original

{ "paths": { "/cards": { "get": { "summary": "Lista todos os cartões", "operationId": "getAllCards", "tags": ["cards-controller"], "parameters": [ { "name": "status", "in": "query", "required": false, "schema": { "type": "string" } } ], "responses": { "200": { "description": "Lista de cartões", "content": { "application/json": { "schema": { "type": "array", "items": { "$ref": "#/components/schemas/Card" } } } } } } } } } }

2. Ferramenta Gerada Automaticamente

Nome da Ferramenta: cards_cards_getAllCards Descrição: [CARDS API - cards-controller] Lista todos os cartões. Parâmetros necessários: status (string, opcional). Em caso de sucesso (HTTP 200), retorna um array de objetos Card. Função: Executa GET /cards com validação de parâmetros e tratamento de erros automático.

3. Uso pela IA

Usuário: "Quais cartões ativos temos no sistema?" IA: Vou consultar os cartões ativos para você. [Executa: cards_cards_getAllCards(status: "active")] Encontrei 15 cartões ativos no sistema: - Cartão Premium (ID: 001) - Status: Ativo - Cartão Gold (ID: 002) - Status: Ativo - Cartão Platinum (ID: 003) - Status: Ativo ...

Por Que Usar o Swagger Agent?

Benefícios práticos para equipes de desenvolvimento que estão iniciando com IA

Produtividade Instantânea

Transforme suas APIs existentes em ferramentas de IA sem reescrever código. Se você já tem especificações OpenAPI, o sistema funciona imediatamente. Isso significa que sua equipe pode começar a experimentar com IA usando a infraestrutura que já possui.

Aprendizado Prático

Este projeto serve como um laboratório educativo completo. Desenvolvedores podem entender na prática como agentes de IA funcionam, como integrar modelos de linguagem em aplicações Java, e como criar interfaces inteligentes que realmente agregam valor.

Democratização da IA

Permite que usuários não-técnicos interajam com sistemas complexos usando linguagem natural. Em vez de precisar conhecer endpoints, parâmetros e formatos JSON, eles podem simplesmente conversar com a IA para obter informações e executar operações.

Arquitetura Escalável

O design modular permite fácil extensão para novos casos de uso. Adicione novos microserviços, modifique prompts, integre diferentes modelos de IA ou customize a interface - tudo foi pensado para crescer junto com as necessidades da sua organização.
